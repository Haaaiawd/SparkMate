# 产品需求文档 (PRD v3.0)

**项目**: 星伴 (SparkMate)
**版本**: v3.0
**日期**: 2026-03-09
**状态**: Active
**负责人**: 研究团队

---

## 1. 产品概述

### 1.1 产品背景

大学生群体面临日益严峻的学业焦虑与孤独感，现有心理健康工具普遍存在门槛高、使用成本大、留存率低等问题。「星伴」以一只名为「微光崽」的情感化几何动画角色为核心，提供无压力的隐形陪伴，同时为教研团队的 SRTP 项目和教研论文收集高质量纵向数据。

### 1.2 产品定位

一款以「微光崽（极简几何呼吸灯形态）」为视觉核心的大学生专注力与情绪追踪微信小程序，兼具**产品趣味性**（养成 + 陪伴 + 无声社交）与**研究严谨性**（纵向自然实验 + 剂量-效应分析）。

### 1.3 目标用户

- **主要用户**: 在校大学生（招募样本 80-120 人，单校园，以班级为单位）
- **管理用户**: 研究团队成员（使用研究后台查看和导出数据）

### 1.4 核心价值主张

> "用无压力的隐形陪伴，抵御校园内的学业焦虑与原子化孤独。"

## 2. 研究框架

> ⚠️ **研究先行原则**：所有功能设计必须服务于研究假设，数据字段在开发前锁定，不得随意增删。

### 2.1 核心研究假设

**主假设 H1**:
> 使用「星伴」系统的学生，使用强度越高（专注打卡频率 × 情绪打卡频率），其情绪状态（PHQ9/GAD7得分下降幅度）改善越显著，学业表现（ΔGPA）也越好。

**辅助假设 H2**:
> AI 主动关怀触发（模板化推送 + LLM 聊天）对情绪改善有显著短期效应（触发后 48h 内情绪滑块得分上升 ≥ 1 分）。

**探索性假设 H3**:
> 特定学业节点（期中/期末考试周）会触发情绪低谷，且该规律在不同专业/年级间存在可统计差异。

### 2.2 研究设计

| 维度 | 设计方案 |
|------|---------|
| **研究类型** | 纵向自然实验（Longitudinal Natural Experiment）|
| **分析方法** | 剂量-效应分析（Dose-Response Analysis）|
| **分组方式** | 无外部对照组；以使用强度自然分组（高频≥5天/周 vs 低频≤2天/周）|
| **样本量目标** | 80-120 人招募；保留有效数据 ≥ 60 人（预留 30% 流失率）|
| **招募渠道** | 班级为单位；德育素拓学时作为激励 |
| **知情同意** | Onboarding 内嵌电子知情同意书（含数据匿名声明和研究目的说明）|

### 2.3 测量时点

| 时点 | 触发条件 | 内容 |
|------|---------|------|
| **T0** | 完成 Onboarding 后立即 | PHQ9 + GAD7 基线量表 |
| **T1** | 入组第 28-32 天（系统自动计算）| PHQ9 + GAD7 中期量表 |
| **T2** | 学期末前 2 周（系统推送 + 导师提醒）| PHQ9 + GAD7 终测量表 |

## 3. 非目标（Non-Goals）

> 明确边界，防止范围蔓延。

1. **不是心理治疗工具**：不提供诊断、不替代专业心理咨询，AI 聊天仅提供情感支持
2. **不是社交平台**：Phase 1 无用户间可识别身份的社交功能，Phase 2 赛博共振保持完全匿名
3. **不做课程管理**：课表预设仅用于场景化触发，不提供课表提醒/任务管理功能
4. **不做成绩分析**：成绩数据由研究团队外部收集，App 内不展示绩点/排名信息
5. **不支持多校园**：MVP 阶段单一校园部署，不做多租户架构

## 4. 用户故事与验收标准

### [REQ-001] 极简冷启动与领养 (Onboarding)

**优先级**: P0
**涉及系统**: miniprogram-system, backend-system, database-system
**用户价值**: 用户建立与微光崽的情感连接，完成研究知情同意

#### 用户故事

- As a 新用户, I want to 快速完成注册并领养我的微光崽, so that 我能开始使用这个陪伴工具
- As a 研究者, I want to 确保每位参与者完成知情同意, so that 数据收集符合伦理规范

#### 验收标准

```gherkin
Given 用户首次打开小程序
When 完成微信授权登录
Then 进入《星伴相处指南》签署页面
  And 展示知情同意书（含数据用途、匿名承诺）
  And 用户点击同意后 consent_given = true 写入数据库

Given 用户完成知情同意
When 进入课表预设步骤
Then 展示一周课程表，支持手动勾选时间块（精度：半小时）
  And 完成后 schedule_slots 写入数据库

Given 用户完成全部 Onboarding 步骤（≤3步）
When 进入主界面
Then 微光崽以"呼吸灯"动画出现在屏幕中央
  And 系统在 24h 内推送 T0 量表（PHQ9 + GAD7）
```

**边界情况**:
- 用户拒绝知情同意 → 弹出提示"无法参与研究版，感谢你的时间"，退出应用
- 用户关闭小程序中途退出课表设置 → 下次进入恢复到课表设置步骤
- 微信授权失败 → 提示"请允许获取昵称，否则无法使用"

### [REQ-002] 专注挂机模式 (Focus Mode)

**优先级**: P0
**涉及系统**: miniprogram-system, backend-system, database-system
**用户价值**: 用户获得专注陪伴体验并积累养成资源；研究者获取 SRL 行为数据

#### 用户故事

- As a 学生, I want to 打开专注模式让微光崽陪我学习, so that 我能感受到一种柔性的自律感
- As a 研究者, I want to 精确记录每次专注的时长和中断次数, so that 量化用户的自我调节学习行为

#### 验收标准

```gherkin
Given 用户点击"开始专注"按钮
When 进入专注模式
Then wx.setKeepScreenOn(true) 生效，屏幕常亮
  And 背景变暗，微光崽化作"专注之火"Lottie动画
  And 计时器开始，显示已专注时长（精度：秒）

Given 用户在专注模式中切出小程序（onHide）
When 切出时长 < 1 分钟
Then 火焰微闪，计时继续
  And distraction_count + 1，distraction_duration 记录实际秒数

Given 用户在专注模式中切出小程序
When 切出时长 ≥ 3 分钟且 < 10 分钟
Then 微光崽火焰熄灭
  And 当次专注经验值衰减 50%
  And long_break_count + 1

Given 用户点击"结束专注"
When 会话时长 ≥ 5 分钟（有效专注最低阈值）
Then 微光崽播放鼓励动画
  And FocusSession 完整写入数据库
  And 养成系统计算经验值，判断是否触发解锁奖励

Given 用户累计专注时长达到解锁阈值
When 阈值触发（如 10h 解锁新光晕颜色）
Then 展示解锁动画
  And 用户可在设置中切换微光崽外观
```

**边界情况**:
- 专注时长 < 5 分钟结束 → 不计入有效 FocusSession，不给经验值，但仍记录数据用于研究
- 手机来电/系统通知导致 onHide → 与普通切出同等处理，不区分
- 专注超过 4 小时未结束 → 自动结束并提示"已连续专注 4 小时，记得休息"

### [REQ-003] 晚点名情绪打卡 (Mood Check-in)

**优先级**: P0
**涉及系统**: miniprogram-system, backend-system, database-system
**用户价值**: 用户以极低成本记录每日情绪；研究者获取高频情绪时序数据

#### 用户故事

- As a 学生, I want to 用 30 秒完成情绪打卡, so that 不会觉得有压力或负担
- As a 研究者, I want to 获得用户每日情绪得分和压力归因, so that 分析情绪基线波动与压力源聚类

#### 验收标准

```gherkin
Given 当前时间在 21:30 - 23:00 区间
When 用户打开小程序
Then 情绪打卡弹出（或主界面显著展示打卡入口）
  And 无级滑块范围 1-10，默认值 5

Given 用户拖动情绪滑块
When 任意值
Then 微光崽颜色实时渐变（1分=灰蓝 #7B9FBB，10分=暖橙 #FF8C42）
  And 渐变流畅，无卡顿（< 16ms 每帧）

Given 用户情绪得分 ≤ 5
When 提交滑块后
Then 微光崽分裂出 4 个可戳气泡：📚学业 / 💬社交 / 🪫身体 / 🌫️莫名
  And 用户点击一个气泡，stress_tag 写入

Given 用户情绪得分 > 5
When 提交滑块后
Then 直接完成打卡，无需选择压力标签
  And stress_tag = null

Given 用户完成打卡
When 写入成功
Then 显示"微光崽收到了你的心跳 🌟"确认动画
  And MoodCheckin 写入数据库（含 week_index, day_of_week）
```

**边界情况**:
- 21:30-23:00 时间窗口外打开 → 不展示情绪打卡入口（时间锁设计）
- 用户同一天多次尝试打卡 → 提示"今天的心跳已记录，明晚再来"，不允许覆盖
- 当天未打卡 → 次日主界面微光崽显示"昨晚没有看到你的心跳，没关系 💙"

### [REQ-004] PHQ9/GAD7 量表系统 (Scale System)

**优先级**: P0
**涉及系统**: miniprogram-system, backend-system, database-system
**用户价值**: 研究者获取标准化心理健康指标，支撑量化研究

#### 用户故事

- As a 研究者, I want to 在 T0/T1/T2 三个时点自动推送量表, so that 获取前后测对比数据
- As a 高风险用户, I want to 在量表得分异常时被及时引导专业资源, so that 获得必要的帮助

#### 验收标准

```gherkin
Given 用户完成 Onboarding
When 24h 内
Then 系统推送 PHQ9（9题）+ GAD7（7题）量表入口
  And 量表以卡片形式展示，Likert 4级单选（0-3分）
  And 预计完成时间 ≤ 5 分钟

Given 用户入组第 28-32 天（以 onboard_at 为基准计算）
When 系统检测到时间窗口
Then 推送 T1 量表并展示提醒气泡

Given 用户完成任意一次量表
When PHQ9 总分 ≥ 15
Then 立即弹出心理援助资源浮层（含学校心理中心电话）
  And high_risk_flag = true 写入数据库
  And 屏蔽 AI 聊天功能（主界面"伙伴聊天"按钮置灰）
  And [ASSUMPTION: 后台发送告警通知给研究团队管理员]

Given 用户完成量表
When 总分 < 15
Then 展示鼓励文字
  And ScaleRecord 写入数据库（保存每题得分 items[]）
```

**边界情况**:
- T1 时间窗口内用户未完成量表 → 每日推送一次提醒，最多推送 5 天
- 用户在量表中途退出 → 不保存部分数据，下次进入从头填写
- T2 时点因学期长度不同而偏移 → 研究管理员可在后台手动设置 T2 时间

### [REQ-005] AI 伙伴聊天（LLM Intervention）

**优先级**: P0
**涉及系统**: miniprogram-system, backend-system, llm-integration
**用户价值**: 用户获得低门槛的情绪倾诉出口；研究者获取干预效果数据

#### 用户故事

- As a 情绪低落的学生, I want to 随时与微光崽聊天, so that 获得非评判性的陪伴感
- As a 研究者, I want to 记录主动/被动触发的聊天和干预效果, so that 分析 AI 干预的响应规律

#### 验收标准

```gherkin
Given 用户 high_risk_flag = false
When 用户点击主界面"伙伴聊天"按钮
Then 进入聊天界面，trigger_type = "PASSIVE"
  And 记录 pre_mood_score（取最近一次 MoodCheckin.mood_score）

Given 后台探针检测到：连续 2 天 mood_score < 4 OR 当周中断率较上周增加 > 50%
When 用户次日首次打开小程序
Then 微光崽形态显示"疲惫/担忧"状态
  And 弹出模板化关怀消息（5条话术模板轮换）
  And 用户点击弹框进入聊天，trigger_type = "PROACTIVE_LOW_MOOD" 或 "PROACTIVE_HIGH_DISTRACTION"

Given 用户在聊天界面发送消息
When 调用 LLM API（DeepSeek，微光崽人设系统提示词）
Then 响应时间 ≤ 3s（p90）
  And LLM 回复符合人设（温暖、好奇、不评判、不给医疗建议）

Given 用户结束聊天（退出聊天界面）
When 结束时
Then 弹出简短情绪复测（单问"现在感觉怎么样" 1-10分滑块）
  And 记录 post_mood_score
  And sentiment_shift_val = post_mood_score - pre_mood_score
  And ChatSession 写入数据库
```

**边界情况**:
- LLM API 超时（> 5s）→ 展示"微光崽在思考中，稍等一下 ✨"，最多等待 10s 后提示重试
- 用户发送敏感内容（自伤相关词汇）→ LLM 系统提示词中设置安全护栏，强制推送心理援助资源
- 每用户每日 LLM 对话轮次上限 20 轮（成本控制）[ASSUMPTION]

### [REQ-006] 微光崽养成系统 (Tamagotchi System)

**优先级**: P1
**涉及系统**: miniprogram-system, backend-system
**用户价值**: 提供长期使用动机，降低用户流失率

#### 用户故事

- As a 学生, I want to 通过持续打卡解锁微光崽的新外观, so that 有动力保持每日使用习惯

#### 验收标准

```gherkin
Given 用户积累有效专注时长
When 达到解锁阈值
Then 解锁对应外观（光晕颜色 / 漂浮特效 / 桌面摆件）
  And 展示解锁动画 + 祝贺文案

Given 用户连续 N 天完成情绪打卡
When 达到连续打卡里程碑（7天 / 30天）
Then 触发特殊外观解锁
```

**养成解锁规则（初始版本）**:

| 条件 | 解锁内容 |
|------|---------|
| 有效专注累计 10h | 光晕颜色：星蓝 |
| 有效专注累计 30h | 漂浮特效：泡泡 |
| 连续情绪打卡 7 天 | 桌面摆件：小书架 |
| 连续情绪打卡 30 天 | 光晕颜色：晨曦橙 |

### [REQ-007] 研究管理后台 (Research Admin Dashboard)

**优先级**: P1
**涉及系统**: admin-system, database-system
**用户价值**: 研究团队监控用户状态、查看数据仪表盘、导出研究数据

#### 用户故事

- As a 研究者, I want to 在 Web 后台查看用户的使用统计, so that 实时监控研究进度
- As a 研究者, I want to 一键导出 CSV 格式数据, so that 导入 SPSS/Python 进行统计分析

#### 验收标准

```gherkin
Given 研究管理员登录后台
When 查看 Dashboard
Then 展示：总用户数 / 今日活跃数 / 平均情绪得分 / 平均专注时长趋势图

Given 研究管理员点击"数据导出"
When 选择时间范围和字段
Then 导出 CSV 文件，包含所有锁定的研究字段（见第5节数据字段表）
  And 导出数据已脱敏（user_id 使用 UUID，无真实姓名/学号）

Given 高风险用户 high_risk_flag 被置为 true
When 系统检测到
Then 后台首页展示高风险用户提醒角标
```

## 6. 非功能性需求

| 维度 | 要求 | 量化标准 |
|------|------|---------|
| **性能** | 小程序冷启动 | < 2s |
| **性能** | 情绪打卡全流程 | ≤ 30s |
| **性能** | LLM 响应 p90 | ≤ 3s |
| **可用性** | 微信版本兼容 | 微信 8.0+ |
| **安全** | 情绪/量表数据 | 加密存储，传输 HTTPS |
| **隐私** | 用户标识 | 前端昵称，后台 UUID，禁止关联真实学号 |
| **伦理** | 高风险处理 | PHQ9 ≥ 15 必须触发资源推送 + 屏蔽 AI 聊天 |
| **数据** | 用户流失保留 | dropout 用户 T0 数据仍保留用于分析 |
